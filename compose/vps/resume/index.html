The day job

i'm an onsite solutions specialist at navanta (formerly bankonit), tier IV escalation for community bank branches across multiple states. when a branch gets built, moved, or breaks in a way nobody else can figure out, i'm the one who shows up.

the stack is regulated banking, end to end: windows server, active directory, group policy, on-prem exchange, plus the switches, firewalls, and VPN tunnels underneath it all. i run new client onboardings from server build to user migration, untangle MFA and hybrid identity messes, and write powershell tooling so the same problem never has to be solved by hand twice.

a lot of the job isn't the servers, though. it's sitting across the table from bank staff: scoping what a branch actually needs, walking owners through what's changing and why, and being the technical face the client calls when it's complicated.

The forge

hart forge is my homelab: a self-hosted platform i run end to end, managed as code. everything below runs on hardware i own plus one small VPS, with configs versioned in my forgejo. it's also where the day-job skills get sharpened: identity, networking, automation, monitoring. same problems, my hardware.

config as code ansible playbooks for fleet health checks, updates, and backups

networking VLAN-segmented unifi LAN, tailscale between sites, cloudflare-only ingress with auto-synced firewall ranges
identity passkey-based OIDC SSO with pocket ID and forward auth
self-hosted git forgejo primary on the VPS with a LAN mirror
monitoring uptime kuma, beszel metrics, and ntfy push alerting
agentic ops a persistent claude-based agent works the fleet with me: ops, deploys, and this site
